Key Observations

  1. The top 3 are folder-level assemblies (BusinessObjects, DataObjects, Utilities) — they function as monolithic shared libraries consumed by virtually everything. Any structural change requires a full solution rebuild.
  2. Logger.cs is the single most-depended-upon individual file at 432 referencing files — more than any other single class in the entire codebase.
  3. The Singleton pattern (SingletonBase.cs) is an architectural cornerstone with 219 inheriting classes, but it creates untestable global state across the entire application.
  4. PTXdsBase.cs and DBManager.cs are the database access bottleneck — every SQL query in the legacy layer transits through these two files.
  5. PTXwfCommon.cs is the UI coupling hub — 259 files depend on it for session/user/menu state, making it impossible to extract any Area independently.
  6. PTXdsCommon.cs has a circular dependency with PTXbsCommon.cs — the DataService layer imports BusinessService, violating the dependency direction rule.
  7. StoredProcedureNames.cs is a single file with 80+ constants — a high-churn file that creates merge conflicts and unnecessary recompilation.
